Given the commits below (defined via their commit message), get the commit id on the current branch and create the respective patch file.

Note that multiple commits may go into one patch. Those commits are separated by "+".
Do not include related test commits into the patches.

1. fix chunking + audio decode -> transcriptions.patch
2. voxtral -> voxtral.patch (ignore "generation config voxtral")
3. usage reporting + cache salting -> anthropic.patch
4. generation config: support frequency_penalty default -> generation_config.patch

The patches must be stored in the repo root.

Patch format requirements:
- Generate plain git diff patches only (files must start with `diff --git`).
- Do NOT use `git format-patch` and do NOT include email-style headers like `From ...`, `Date:`, `Subject: [PATCH]`, or `---` shortstat preamble.
- Use patch output compatible with `git apply`.
